# Featured Image Prompts for "When Everything Fails"

## Article Context
- **Title**: When Everything Fails: The Art of Failing Gracefully
- **Series**: Resilience Engineering Part 4
- **Theme**: Graceful degradation, cascading failures, resilience patterns
- **Target**: Software architects, SREs, platform engineers

## Gemini Image Generation Prompts

### Option 1: Abstract Technical Metaphor
```
Create a modern, minimalist technical illustration showing a network of interconnected nodes with some nodes glowing red (failing) while others remain blue (healthy), with protective barriers or shields around certain clusters. The image should convey the concept of graceful degradation - some parts failing while the system continues to function. Use a dark background with vibrant blue and red accent colors. Style: clean, geometric, professional tech illustration suitable for a Medium article about system resilience. 16:9 aspect ratio.
```

**Focus**: Network/distributed systems angle, visual representation of partial failure

### Option 2: Dramatic Failure Cascade
```
Create a dramatic isometric illustration of a multi-tier system architecture where the top layers are crumbling or breaking apart (showing failure) but the foundation layers remain solid and intact. Show circuit breaker symbols, warning indicators, and data flow paths being rerouted around failed components. Use a color scheme of dark blues, oranges for warnings, and reds for failures. The image should convey "when everything fails" but with controlled degradation. Style: modern tech illustration with depth and dimension. 16:9 aspect ratio.
```

**Focus**: Dramatic "everything fails" scenario with controlled degradation, architectural layers

### Option 3: Resilience Through Layers
```
Create a sleek technical diagram showing a pyramid or layered hierarchy with 5 distinct levels, where the top levels are partially transparent or faded (degraded) while the bottom levels remain solid and bright. Include visual elements like shields, fallback arrows, and circuit breaker icons. The image should represent the degradation hierarchy concept - features shedding gracefully from experimental down to static content. Use a professional color palette of deep blues, teals, and subtle orange accents. Style: clean, modern infographic suitable for enterprise tech content. 16:9 aspect ratio.
```

**Focus**: Degradation hierarchy design principle, capability levels (static → experimental)

## Design Guidelines

**Color Palette**:
- Primary: Deep blues, teals
- Accent: Orange (warnings), Red (failures)
- Background: Dark (for contrast)
- Healthy states: Blue/teal
- Degraded states: Orange/amber
- Failed states: Red

**Visual Elements**:
- Circuit breaker symbols
- Shield/barrier icons (protection)
- Network nodes and connections
- Layered architecture
- Data flow arrows
- Warning indicators

**Style Requirements**:
- Professional, enterprise-appropriate
- Clean, modern aesthetic
- Technical but accessible
- 16:9 aspect ratio for Medium featured image
- High contrast for readability

## Selection Criteria

Choose based on:
1. **Visual impact**: Which catches attention in Medium feed?
2. **Concept clarity**: Which best represents graceful degradation?
3. **Professional appeal**: Which resonates with target audience (architects, SREs)?
4. **Brand consistency**: Which aligns with other articles in resilience series?
