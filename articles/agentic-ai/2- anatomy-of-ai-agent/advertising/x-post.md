## Thread

### Tweet 2

THE AGENT LOOP

Every agent operates on a cycle:
→ Perceive (observe environment)
→ Reason (understand context)
→ Plan (formulate strategy)
→ Act (execute with tools)
→ Observe (evaluate results)

This loops until the goal is achieved.

### Tweet 3

PLANNING

Agents break down complex goals into subtasks.

"Book a vacation to Japan" becomes:
1. Research destinations
2. Check flights
3. Find accommodation
4. Create itinerary
5. Make reservations

Each subtask decomposes further until actionable.

### Tweet 4

MEMORY

Short-term: Current task context (what's happening now)
Long-term: Past experiences, learned patterns, user preferences

Like humans, agents remember what worked before and apply those lessons to new problems.

Vector databases enable semantic memory search.

### Tweet 5

TOOL USE

Language models generate text. Tools let agents ACT.

Tools = APIs, databases, external services

Example: Financial analysis agent uses:
- web_search()
- extract_financial_data()
- query_database()
- calculate_ratios()
- generate_report()

### Tweet 6

TOOL ORCHESTRATION

Agents don't just use tools—they compose them:

Sequential: search → extract → analyze → report
Parallel: search financials + search news + search competitors
Conditional: if complaint → escalate, if question → search KB

### Tweet 7

REAL EXAMPLE

Task: "Evaluate investment in Company X"

Agent:
→ Plans: gather data, analyze market, assess risks
→ Remembers: similar investments, criteria, patterns
→ Uses tools: web search, data extraction, calculations
→ Delivers: comprehensive analysis in minutes
