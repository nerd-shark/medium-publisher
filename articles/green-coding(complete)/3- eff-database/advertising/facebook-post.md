Your database is probably your biggest energy hog. Disk I/O is 1000x slower than memory. Full table scans waste CPU and disk.

Part 3 of my Green Coding series.

Key insights:

✓ Index strategically (B-tree for ranges, hash for equality)
✓ Connection pooling (reuse connections)
✓ Query result caching (Redis, Memcached)
✓ Batch operations (bulk inserts)
✓ Avoid SELECT * (fetch only needed columns)
✓ Real case: 15s product search → 120ms with indexes
