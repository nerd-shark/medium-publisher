Your database is probably your biggest energy hog. Disk I/O is 1000x slower than memory. Full table scans waste CPU and disk.

Part 3 of my Green Coding series.

1️⃣ Index strategically (B-tree for ranges, hash for equality)
2️⃣ Connection pooling (reuse connections)
3️⃣ Query result caching (Redis, Memcached)
4️⃣ Batch operations (bulk inserts)
5️⃣ Avoid SELECT * (fetch only needed columns)
6️⃣ Real case: 15s product search → 120ms with indexes
